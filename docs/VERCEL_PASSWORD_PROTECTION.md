# Vercel のパスワード保護のやり方

Vercel の「Deployment Protection → Password Protection」を使うと、**デプロイされたサイトにパスワードをかける**ことができます。設定だけで、コードを書かずに保護できます。

---

## ⚠️ 大事な前提：保護できる範囲

**Vercel のパスワード保護は「サイト全体」にしかかけられません。**

- ✅ **できること**: デプロイ（本番 or プレビュー）にアクセスする**すべての URL** で、最初に1回パスワードを入力させる。
- ❌ **できないこと**: 「`/admin` だけパスワード」「トップページは誰でも」のように**パスごとに分ける**ことは、Vercel の機能だけではできません。

そのため、次のどちらかに当てはまる場合に向いています。

| 使い方 | 向いている人 |
|--------|----------------|
| **本番サイト全体をパスワードで守りたい** | まだ公開前の「お試し本番」や、関係者だけに見せたいとき |
| **プレビュー（Preview）環境だけ守りたい** | 本番は公開のまま、PR のプレビュー URL だけパスワードをかけたいとき |

**「お客様はトップページは見れるようにして、管理画面（`/admin`）だけ守りたい」** 場合は、Vercel のこの機能では実現できません。そのときは **Basic 認証（A）** のミドルウェアを使う必要があります。

---

## 必要なプラン

- **Password Protection** は **Vercel の Pro または Enterprise のチームプラン** で利用できます。
- Hobby（無料）プランでは使えません。

---

## 手順：Vercel でパスワード保護を有効にする

### 1. Vercel ダッシュボードを開く

1. ブラウザで [https://vercel.com/dashboard](https://vercel.com/dashboard) を開く。
2. ログインして、**対象のプロジェクト**（この Next.js アプリ）をクリックする。

### 2. 設定画面の「Deployment Protection」を開く

1. 画面上方の **「Settings」** タブをクリック。
2. 左メニューから **「Deployment Protection」** をクリック。

### 3. Password Protection をオンにする

1. **「Password Protection」** のセクションを探す。
2. **トグルをオン（Enable）** にする。
3. **どの環境を守るか** を選ぶ：
   - **Production** … 本番の URL（`https://あなたのドメイン.vercel.app` など）を保護。
   - **Preview** … PR やブランチからできるプレビュー URL だけ保護。
   - **All** … 本番もプレビューも両方保護。
4. **パスワード** を入力する（任意の文字列。推測されにくい長いもの推奨）。
5. 最後に **「Save」** をクリック。

### 4. 動作確認

1. 保護した環境の URL を、**シークレットウィンドウなど別タブ** で開く。
2. **パスワード入力画面** が表示され、正しいパスワードを入れるとサイトが見えれば OK。
3. 同じブラウザでは、しばらくは Cookie で記憶されるので、再度パスワードを聞かれないことがあります。

---

## まとめ

| やりたいこと | 使うもの |
|--------------|----------|
| **サイト全体**（本番 or プレビュー）にパスワードをかけたい | このガイドのとおり、Vercel の Password Protection を有効にする。 |
| **`/admin` だけ** パスワードで守り、トップは誰でも見れるようにしたい | Vercel のこの機能では不可。`docs/NEXT_STEPS_ADMIN_AUTH.md` の **Basic 認証（A）** を使う。 |

Vercel の設定は **Project → Settings → Deployment Protection → Password Protection** で行います。Pro/Enterprise チームであれば、上記の手順でパスワード保護を有効にできます。
