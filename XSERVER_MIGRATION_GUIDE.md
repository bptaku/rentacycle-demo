1. **ドメイン管理はXサーバーのまま**

   - 開発に集中できる

4. **技術的負担**
   - システム変更不要
   - すぐに運用開始可能

## 🔧 実装方法：XサーバーのドメインをVercelで使う

### 手順1: Vercelにアプリをデプロイ

1. Vercelアカウントを作成（無料）
2. GitHubリポジトリを連携
3. 環境変数を設定
4. デプロイ（自動で `https://your-app.vercel.app` が発行される）

### 手順2: XサーバーでDNS設定を変更

Xサーバーのドメイン管理画面で、以下のDNSレコードを追加/変更：

```
# カスタムドメインをVercelに設定する場合
A レコード（または CNAME レコード）:
  @ (または www) → Vercelが指定するIPアドレス
```

**具体的な手順:**

1. Vercel Dashboard → プロジェクト → Settings → Domains
2. カスタムドメインを追加（例: `yourdomain.com`）
3. Vercelが表示するDNS設定をコピー
4. Xサーバーのドメイン管理画面に移動
5. DNS設定で、Vercelが指定した設定を追加

**例:**
```
A レコード:
  @ → 76.76.21.21

CNAME レコード:
  www → cname.vercel-dns.com
```

### 手順3: 環境変数の設定

Vercel Dashboardで `NEXT_PUBLIC_BASE_URL` を設定：

```
NEXT_PUBLIC_BASE_URL = https://yourdomain.com
```

これで、メール内のキャンセル申請リンクが正しいURLになります。

## 🔄 XサーバーVPSを選ぶ場合の設定手順

もしXサーバーVPSを選ぶ場合、以下の手順が必要です：

### 1. VPSサーバーの準備

```bash
# サーバーにSSH接続
ssh your-user@your-server-ip
```

### 2. Node.jsのインストール

```bash
# Node.js 20をインストール（例: nvmを使用）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 20
nvm use 20
```

### 3. アプリケーションのデプロイ

```bash
# リポジトリをクローン
git clone https://github.com/your-username/rentacycle-demo.git
cd rentacycle-demo

# 依存関係をインストール
npm install

# ビルド
npm run build

# 環境変数を設定
cp env.example .env.local
# .env.localを編集して環境変数を設定
```

### 4. PM2でプロセス管理

```bash
# PM2をインストール
npm install -g pm2

# アプリを起動
pm2 start npm --name "rentacycle" -- start

# 自動起動を設定
pm2 startup
pm2 save
```

### 5. Nginxの設定（リバースプロキシ）

```bash
# Nginxをインストール
sudo yum install nginx  # CentOS系の場合
# または
sudo apt install nginx  # Ubuntu系の場合

# Nginx設定ファイルを作成
sudo nano /etc/nginx/sites-available/rentacycle
```

Nginx設定例:
```nginx
server {
    listen 80;
    server_name yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 6. SSL証明書の設定（Let's Encrypt）

```bash
# Certbotをインストール
sudo yum install certbot python3-certbot-nginx

# SSL証明書を取得
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

### 7. 自動デプロイの設定（GitHub Actionsなど）

GitHub Actionsで自動デプロイを設定するか、手動でデプロイするか選択。

## 📊 「予想トラフィック」とは？

**予想トラフィック = システムが処理する予約数やアクセス数**

レンタサイクル事業の場合、以下のような目安です：

### 小規模（月100件以下）
- 1日の予約数: 1〜3件
- 1ヶ月の予約数: 30〜100件
- 1年: 500〜1,000件程度
- **→ Vercel Hobby（無料）で十分**

### 中規模（月1,000件以下）
- 1日の予約数: 10〜30件
- 1ヶ月の予約数: 300〜1,000件
- 1年: 5,000〜10,000件程度
- **→ Vercel Pro（月3,000円）が安心**

### 大規模（月1,000件以上）
- 1日の予約数: 30件以上
- 1ヶ月の予約数: 1,000件以上
- 1年: 10,000件以上
- **→ Vercel Pro または XサーバーVPS**

## 🎯 あなたの事業規模の分析

### ケース1: 月67件の予約（小規模）

- **繁忙期の月**: 200台（自転車の貸出数）
- **予約数**: 約67件/月（200台 ÷ 3）
- **年間予想**: 約500〜800件程度

**規模の判定**: ✅ **小規模** → **Vercel Hobby（無料プラン）で十分対応可能**

| 項目 | 制限 | 使用量（予想） | 判定 |
|------|------|--------------|------|
| **Serverless Function実行** | 100回/日 | 約2〜3回/日 | ✅ 余裕あり |
| **帯域幅** | 100GB/月 | 約5〜10GB/月 | ✅ 余裕あり |
| **ビルド時間** | 6,000分/月 | 約100〜200分/月 | ✅ 余裕あり |

**結論: 無料プランで問題なく運用できます！**

---

### ケース2: 月200件の予約（中規模）

- **予約数**: 200件/月
- **年間予想**: 約2,000〜2,400件程度

**規模の判定**: ⚠️ **中規模** → **Vercel Hobby（無料プラン）で対応可能だが、Proプランも検討**

#### Vercel Hobbyプランでの使用量予想

| 項目 | 制限 | 使用量（予想） | 判定 |
|------|------|--------------|------|
| **Serverless Function実行** | 100回/日 | 約7〜10回/日 | ✅ 余裕あり |
| **帯域幅** | 100GB/月 | 約20〜30GB/月 | ✅ 余裕あり |
| **ビルド時間** | 6,000分/月 | 約200〜400分/月 | ✅ 余裕あり |

**結論: 無料プランでも運用可能ですが、余裕を持たせるならProプランも検討**

#### 注意点

1. **在庫確認APIの呼び出し回数**
   - 予約フォームで在庫確認が頻繁に呼ばれる可能性
   - 1回の予約作成で複数回の在庫確認APIが呼ばれる
   - デバウンス（入力待ち）機能があるので、実際の呼び出しは少ない

2. **管理画面のアクセス**
   - 管理者が頻繁にアクセスする場合、API呼び出しが増える
   - それでも100回/日の制限内に収まる見込み

3. **繁忙期の集中**
   - 繁忙期に予約が集中しても、1日あたりの平均は約7件
   - ピーク時でも20〜30件/日程度 → 100回/日の制限内

#### 推奨

- **まずは無料プランで運用開始**
- **月200件を超える、または制限に近づいたらProプランにアップグレード**
- **Proプラン（月3,000円）でも十分コストパフォーマンスが良い**

## 🌐 既存のXサーバーでのウェブサイトについて

既にXサーバーでウェブサイトを構築している場合、以下の選択肢があります：

### 選択肢A: サブドメインで分ける

**既存サイト:** `https://yourdomain.com`（Xサーバーで運用継続）  
**予約システム:** `https://reserve.yourdomain.com`（Vercelで運用）

**メリット:**
- ✅ 既存サイトに影響なし
- ✅ 簡単に分離できる
- ✅ 管理が分かりやすい

**設定方法:**
1. Xサーバーのドメイン管理画面で、サブドメインを追加
2. DNS設定で `reserve` サブドメインをVercelに設定
3. Vercelでカスタムドメイン `reserve.yourdomain.com` を設定

---

### 選択肢B: パスで分ける

**既存サイト:** `https://yourdomain.com/`（Xサーバー）  
**予約システム:** `https://yourdomain.com/reserve`（Vercel）

**注意:** この方法は複雑で、リバースプロキシの設定が必要です。**おすすめしません。**

---

### 選択肢C: 別ドメインを使う

**既存サイト:** `https://yourdomain.com`（Xサーバー）  
**予約システム:** `https://rentacycle.yourdomain.com`（Vercel）

新しいドメインを取得するか、サブドメインを使用。
