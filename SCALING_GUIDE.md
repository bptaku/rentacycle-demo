# スケーリングガイド（月200件の予約規模向け）

## 📊 月200件の予約規模での分析

### 予約状況
- **予約数**: 200件/月
- **1日の平均**: 約7件/日
- **繁忙期のピーク**: 20〜30件/日程度
- **年間予想**: 約2,000〜2,400件

### 規模の判定
⚠️ **中規模** → 無料プランでも運用可能だが、Proプランも検討の余地あり

## 📈 使用量の詳細分析

### API呼び出し回数の内訳（月200件の予約）

#### 予約作成関連
- **予約作成API**: 200回/月 = 約7回/日
- **在庫確認API**: 約600〜1,000回/月 = 約20〜30回/日
  - 1回の予約作成で、複数車種の在庫確認が発生
  - デバウンス機能により、実際の呼び出しは抑えられる

#### 管理画面関連
- **予約一覧取得API**: 約100〜200回/月 = 約3〜7回/日
- **在庫管理API**: 約50〜100回/月 = 約2〜3回/日
- **ステータス更新API**: 約200〜400回/月 = 約7〜13回/日

#### 合計
- **1日の合計**: 約40〜60回/日
- **Vercel Hobbyの制限**: 100回/日
- **余裕**: 約40〜60回/日

**結論**: 無料プランでも十分運用可能

## 🎯 推奨アプローチ

### 段階的な運用戦略

#### フェーズ1: 無料プランで開始（推奨）

1. **Vercel Hobby（無料）で運用開始**
2. **1〜2ヶ月様子を見る**
3. **使用量を監視**
   - Vercel Dashboard → Analytics で使用量を確認
   - 制限に近づいていないか確認

#### フェーズ2: アップグレード判断

**以下の条件に当てはまる場合、Proプランにアップグレードを検討:**

1. ✅ 月200件を安定して超える
2. ✅ 使用量が制限の70%を超える
3. ✅ パフォーマンスの向上が必要
4. ✅ 詳細な分析機能が必要

**アップグレードのタイミング:**
- 使用量が制限の80%を超えたら検討
- または、予約数が月250件を超えたら検討
